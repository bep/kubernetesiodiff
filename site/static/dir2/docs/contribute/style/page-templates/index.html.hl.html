    <h1>
      Using Page Templates
    </h1>
    <p>
      <a href="https://github.com/kubernetes/website/edit/master/content/en/docs/contribute/style/page-templates.md" id="editPageButton" target="_blank">
        Edit This Page
      </a>
    </p>
    <h1>
      Using Page Templates
    </h1>
    <p>
      When contributing new topics, apply one of the following templates to them.
      This standardizes the user experience of a given page.
    </p>
    <p>
      The page templates are in the
      <a href="https://git.k8s.io/website/layouts/partials/templates">
        <code>
          layouts/partials/templates
        </code>
      </a>
      directory of the
      <a href="https://github.com/kubernetes/website">
        <code>
          kubernetes/website
        </code>
      </a>
      repository.
    </p>
    <blockquote class="note">
      <div>
        <strong>
          Note:
        </strong>
        Every new topic needs to use a template. If you are unsure which
        template to use for a new topic, start with the
        <a href="#concept-template">
          concept template
        </a>
        .
      </div>
    </blockquote>
    <ul id="markdown-toc">
      <li>
        <a href="#concept-template">
          Concept template
        </a>
      </li>
      <li>
        <a href="#task-template">
          Task template
        </a>
      </li>
      <li>
        <a href="#tutorial-template">
          Tutorial template
        </a>
      </li>
      <li>
        <a href="#what-s-next">
          What's next
        </a>
      </li>
    </ul>
    <h2 id="concept-template">
      Concept template
    </h2>
    <p>
      A concept page explains some aspect of Kubernetes. For example, a concept
      page might describe the Kubernetes Deployment object and explain the role it
      plays as an application once it is deployed, scaled, and updated. Typically, concept
      pages don&rsquo;t include sequences of steps, but instead provide links to tasks or
      tutorials.
    </p>
    <p>
      To write a new concept page, create a Markdown file in a subdirectory of the
      <code>
        /content/en/docs/concepts
      </code>
      directory, with the following characteristics:
    </p>
    <ul>
      <li>
        <p>
          In the page&rsquo;s YAML front-matter, set
          <code>
            content_template: templates/concept
          </code>
          .
        </p>
      </li>
      <li>
        <p>
          In the page&rsquo;s body, set the required
          <code>
            capture
          </code>
          variables and any optional
          ones you want to include:
        </p>
        <table>
          <thead>
            <tr>
              <th>
                Variable
              </th>
              <th>
                Required?
              </th>
            </tr>
          </thead>
          <tbody>
            <tr>
              <td>
                overview
              </td>
              <td>
                yes
              </td>
            </tr>
            <tr>
              <td>
                body
              </td>
              <td>
                yes
              </td>
            </tr>
            <tr>
              <td>
                whatsnext
              </td>
              <td>
                no
              </td>
            </tr>
          </tbody>
        </table>
        <p>
          The page&rsquo;s body will look like this (remove any optional captures you don&rsquo;t
          need):
        </p>
        <pre><code>{{% capture overview %}}

{{% /capture %}}

{{% capture body %}}

{{% /capture %}}

{{% capture whatsnext %}}

{{% /capture %}}</code></pre>
      </li>
      <li>
        <p>
          Fill each section with content. Follow these guidelines:
        </p>
        <ul>
          <li>
            Organize content with H2 and H3 headings.
          </li>
          <li>
            For
            <code>
              overview
            </code>
            , set the topic&rsquo;s context with a single paragraph.
          </li>
          <li>
            For
            <code>
              body
            </code>
            , explain the concept.
          </li>
          <li>
            For
            <code>
              whatsnext
            </code>
            , provide a bulleted list of topics (5 maximum) to learn more about the concept.
          </li>
        </ul>
      </li>
    </ul>
    <p>
      <a href="/docs/concepts/overview/working-with-objects/annotations/">
        Annotations
      </a>
      is a published example of the concept template. This page also uses the concept template.
    </p>
    <h2 id="task-template">
      Task template
    </h2>
    <p>
      A task page shows how to do a single thing, typically by giving a short
      sequence of steps. Task pages have minimal explanation, but often provide links
      to conceptual topics that provide related background and knowledge.
    </p>
    <p>
      To write a new task page, create a Markdown file in a subdirectory of the
      <code>
        /content/en/docs/tasks
      </code>
      directory, with the following characteristics:
    </p>
    <ul>
      <li>
        <p>
          In the page&rsquo;s YAML front-matter, set
          <code>
            content_template: templates/task
          </code>
          .
        </p>
      </li>
      <li>
        <p>
          In the page&rsquo;s body, set the required
          <code>
            capture
          </code>
          variables and any optional
          ones you want to include:
        </p>
        <table>
          <thead>
            <tr>
              <th>
                Variable
              </th>
              <th>
                Required?
              </th>
            </tr>
          </thead>
          <tbody>
            <tr>
              <td>
                overview
              </td>
              <td>
                yes
              </td>
            </tr>
            <tr>
              <td>
                prerequisites
              </td>
              <td>
                yes
              </td>
            </tr>
            <tr>
              <td>
                steps
              </td>
              <td>
                no
              </td>
            </tr>
            <tr>
              <td>
                discussion
              </td>
              <td>
                no
              </td>
            </tr>
            <tr>
              <td>
                whatsnext
              </td>
              <td>
                no
              </td>
            </tr>
          </tbody>
        </table>
        <p>
          The page&rsquo;s body will look like this (remove any optional captures you don&rsquo;t
          need):
        </p>
        <pre><code>{{% capture overview %}}

{{% /capture %}}

{{% capture prerequisites %}}

{{&lt; include &quot;task-tutorial-prereqs.md&quot; &gt;}} {{&lt; version-check &gt;}}

{{% /capture %}}

{{% capture steps %}}

{{% /capture %}}

{{% capture discussion %}}

{{% /capture %}}

{{% capture whatsnext %}}

{{% /capture %}}</code></pre>
      </li>
      <li>
        <p>
          Within each section, write your content. Use the following guidelines:
        </p>
        <ul>
          <li>
            Use a minimum of H2 headings (with two leading
            <code>
              #
            </code>
            characters). The sections
            themselves are titled automatically by the template.
          </li>
          <li>
            For
            <code>
              overview
            </code>
            , use a paragraph to set context for the entire topic.
          </li>
          <li>
            For
            <code>
              prerequisites
            </code>
            , use bullet lists when possible. Start adding additional
            prerequisites below the
            <code>
              include
            </code>
            . The default prerequisites include a running Kubernetes cluster.
          </li>
          <li>
            For
            <code>
              steps
            </code>
            , use numbered lists.
          </li>
          <li>
            For discussion, use normal content to expand upon the information covered
            in
            <code>
              steps
            </code>
            .
          </li>
          <li>
            For
            <code>
              whatsnext
            </code>
            , give a bullet list of up to 5 topics the reader might be
            interested in reading next.
          </li>
        </ul>
      </li>
    </ul>
    <p>
      An example of a published topic that uses the task template is
      <a href="/docs/tasks/access-kubernetes-api/http-proxy-access-api">
        Using an HTTP proxy to access the Kubernetes API
      </a>
      .
    </p>
    <h2 id="tutorial-template">
      Tutorial template
    </h2>
    <p>
      A tutorial page shows how to accomplish a goal that is larger than a single
      task. Typically a tutorial page has several sections, each of which has a
      sequence of steps. For example, a tutorial might provide a walkthrough of a
      code sample that illustrates a certain feature of Kubernetes. Tutorials can
      include surface-level explanations, but should link to related concept topics
      for deep explanations.
    </p>
    <p>
      To write a new tutorial page, create a Markdown file in a subdirectory of the
      <code>
        /content/en/docs/tutorials
      </code>
      directory, with the following characteristics:
    </p>
    <ul>
      <li>
        <p>
          In the page&rsquo;s YAML front-matter, set
          <code>
            content_template: templates/tutorial
          </code>
          .
        </p>
      </li>
      <li>
        <p>
          In the page&rsquo;s body, set the required
          <code>
            capture
          </code>
          variables and any optional
          ones you want to include:
        </p>
        <table>
          <thead>
            <tr>
              <th>
                Variable
              </th>
              <th>
                Required?
              </th>
            </tr>
          </thead>
          <tbody>
            <tr>
              <td>
                overview
              </td>
              <td>
                yes
              </td>
            </tr>
            <tr>
              <td>
                prerequisites
              </td>
              <td>
                yes
              </td>
            </tr>
            <tr>
              <td>
                objectives
              </td>
              <td>
                yes
              </td>
            </tr>
            <tr>
              <td>
                lessoncontent
              </td>
              <td>
                yes
              </td>
            </tr>
            <tr>
              <td>
                cleanup
              </td>
              <td>
                no
              </td>
            </tr>
            <tr>
              <td>
                whatsnext
              </td>
              <td>
                no
              </td>
            </tr>
          </tbody>
        </table>
        <p>
          The page&rsquo;s body will look like this (remove any optional captures you don&rsquo;t
          need):

        <pre><code>{{% capture overview %}}

{{% /capture %}}

{{% capture prerequisites %}}

{{&lt; include &quot;task-tutorial-prereqs.md&quot; &gt;}} {{&lt; version-check &gt;}}

{{% /capture %}}

{{% capture objectives %}}

{{% /capture %}}

{{% capture lessoncontent %}}

{{% /capture %}}

{{% capture cleanup %}}

{{% /capture %}}

{{% capture whatsnext %}}

{{% /capture %}}</code></pre>
      </li>
      <li>
        <p>
          Within each section, write your content. Use the following guidelines:
        </p>
        <ul>
          <li>
            Use a minimum of H2 headings (with two leading
            <code>
              #
            </code>
            characters). The sections
            themselves are titled automatically by the template.
          </li>
          <li>
            For
            <code>
              overview
            </code>
            , use a paragraph to set context for the entire topic.
          </li>
          <li>
            For
            <code>
              prerequisites
            </code>
            , use bullet lists when possible. Add additional
            prerequisites below the ones included by default.
          </li>
          <li>
            For
            <code>
              objectives
            </code>
            , use bullet lists.
          </li>
          <li>
            For
            <code>
              lessoncontent
            </code>
            , use a mix of numbered lists and narrative content as
            appropriate.
          </li>
          <li>
            For
            <code>
              cleanup
            </code>
            , use numbered lists to describe the steps to clean up the
            state of the cluster after finishing the task.
          </li>
          <li>
            For
            <code>
              whatsnext
            </code>
            , give a bullet list of up to 5 topics the reader might be
            interested in reading next.
          </li>
        </ul>
      </li>
